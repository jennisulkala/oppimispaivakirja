Tällä viikolla käytiin läpi sitä miten kuvia voi optimoida käytettäväksi netissä, jotta ne ei ois niin raskaita. Puhuttiin sekä siitä miten itse kuvasta voi esim Photarilla tehdä kevyemmän vaikkapa halftonea käyttämällä, mutta myös käyttämällä netissä olevia konverttereita (esim. TinyPNG), jotka pakkaa niin kuvista, videoista kuin audioistakin pienempiä tiedostoja. Opin myös, että vektoritiedostoja voi tuoda koodimuodossa suoraan Illustratorista, jolla saa tosi teräviä "kuvia" aikaan. Koodin voi tuoda klikkaamalla Export As > SVG > Show Code. Myöhemmin tätä koodia voi myös editoida suoraan koodin sisällä, eikä kuvaa tarvitse esimerkiksi mennä muuttamaan Illustratoriin.

Puhuttiin CSS:n (Cascading Style Sheets) tärkeimmistä peruselementeistä, jotka pitäisi oppia, jotta toimivia nettisivuja pystyy rakentamaan. Tärkeimpiä konsepteja ovat box-model, cascade, inheritance ja specifity. Puhuttiin siis vähän grideistä ja luin CSS-Tricks-sivustolta aiheesta vähän lisää ja luulen ymmärtäneeni miten sen perusteet suunnilleen toimii.

Grideillä pystyy rakentamaan siis tosi monimutkaisiakin leiskoja ja sillä voi rakentaa koko sivuston pohjan.

Cascadella taas tarkoitetaan käytännössä sitä, että CSS-sääntöjen järjestyksellä on väliä. Jos koodissa on peräkkäin esimerkiksi kaksi samanarvoista CSS-sääntöä, on jälkimmäinen aina se, joka "määrää lopputuloksen". Eli esimerkiksi näin:

Specifityllä taas tarkoitetaan sitä, että jos jokin CSS-sääntö on spesifimpi kuin toinen, vaikka ne olisivat CSS-koodin järjestyksessä ns. väärin päin, kumoaa spesifimpi sääntö vähemmän spesifin säännön. Specifity järjestys: browser > element > class > id. Id:llä spesifiointia ei kuitenkaan suositella käyttämään, sillä se on niin spesifi, että se saattaa aiheuttaa helposti ongelmia. Se toimii kuitenkin joihinkin tiettyihin funktioihin todella hyvin, esim. tilanteisiin, joissa haluaa vaikkapa klikata "takaisin ylös", jota monissa sivuissa käytetään. Tässä selitetään hyvin se, miksi sitä ei kuitenkaan yleisesti ottaen kannattaisi käyttää:

"IDs should be unique on a page. This means that if you attach a style to an ID, you won't be able to reuse it within the same webpage. Classes, however, can appear on several HTML elements on the same page. Being able to reuse styles is one of the advantages of CSS. For example, it would be difficult to style a list element directly using just IDs. Admittedly, not every style needs to be reused, but it's nice to have the option." Teksti täältä.

Inheritance-konseptilla taas tarkoitaan sitä, että osa CSS:ään asetetuista parent elementtien arvoista periytyy niiden child elementeille. On tärkeää kuitenkin huomata, että kaikki arvot eivät kuitenkaan periydy kaikille elementeille.

Puhuttiin myös aiheesta flex wrap ja flex basis sekä CSS Culc ja Variable -funktioista, mutta niihin mun pitää vielä paneutua lisää kun aivot on enemmän vastaanottavaisia.
